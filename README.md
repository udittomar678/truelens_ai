## What Is TrueLens AI?

TrueLens AI is a forensic intelligence platform that detects whether an image was **generated by an AI** (Stable Diffusion, Midjourney, DALL-E, GANs) or **taken by a real camera**.

It combines three independent analysis signals into a single weighted verdict:

| Signal | Method | Weight |
|--------|--------|--------|
| 🧠 Deep Learning | EfficientNet-B3 CNN analyzes pixel patterns and textures | 65% |
| 📷 Metadata Forensics | EXIF data inspection for missing camera info | 20% |
| 📡 Frequency Analysis | FFT spectrum detects invisible GAN/diffusion artifacts | 15% |

Results are explained visually using **Grad-CAM heatmaps** — showing exactly which regions of the image triggered the AI detection signal.

- ✅ **90.82% accuracy** on CIFAKE benchmark (120,000 images)

- ✅ **Web interface** — dark forensic UI with drag-and-drop upload
- ✅ **Sub-100ms inference** — full pipeline per image
- ✅ **Apple MPS support** — GPU acceleration on Apple Silicon

## Architecture

```
INPUT IMAGE (224×224×3)
        │
        ├─────────────────────────────────┐
        ▼                                 ▼
  SPATIAL BRANCH                  FREQUENCY BRANCH
  EfficientNet-B3                 FFT Transform
  385 layers · 10.7M params       + 3-layer MLP
  Detects: textures, edges,       Detects: GAN artifacts,
  unnatural smoothness            diffusion signatures
        │                                 │
        ▼                                 ▼
   512 features                    64 features
        │                                 │
        └──────────────┬──────────────────┘
                       ▼
              FUSION CLASSIFIER
              576 → 512 → 256 → 2
                       │
                       ▼
               AI PROBABILITY
               VERDICT + HEATMAP
```

## Installation

### Prerequisites

- Python 3.11+
- pip

### Setup

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/truelens_ai.git
cd truelens_ai

# Create virtual environment
python3 -m venv venv
source venv/bin/activate       # Mac/Linux
# venv\Scripts\activate        # Windows

# Install dependencies
pip install -r requirements.txt
```

### Download Model Weights

The trained weights file (`truelens_image.pth`) is not included in the repo due to file size. You can either:

**Option A — Train from scratch:**
```bash
# Download CIFAKE dataset and train
python train.py --data data/ --epochs 30
```

**Option B — Download pre-trained weights:**

Place `truelens_image.pth` in the `weights/` folder.

## Training

### Dataset

TrueLens was trained on [CIFAKE](https://www.kaggle.com/datasets/birdy654/cifake-real-and-ai-generated-synthetic-images) — 120,000 images:

| Split | REAL | FAKE | Total |
|-------|------|------|-------|
| Train | 50,000 | 50,000 | 100,000 |
| Val | 10,000 | 10,000 | 20,000 |

### Prepare Data

```
data/
├── train/
│   ├── REAL/    ← real images
│   └── FAKE/    ← AI generated images
└── val/
    ├── REAL/
    └── FAKE/
```

### Run Training

```bash
# Quick test (3 epochs)
python train.py --data data/ --epochs 3

# Full training
python train.py --data data/ --epochs 30

# With larger batch size (faster on M4)
python train.py --data data/ --batch-size 64
```

### Training Results

| Epoch | Val Loss | Val Accuracy |
|-------|----------|--------------|
| 1 | 0.287 | 87.3% |
| 2 | 0.234 | 89.1% |
| **3** | **0.218** | **90.82% ← best** |
| 4–8 | 0.223+ | 90.4% ↓ |

Early stopping triggered at epoch 8.

## How Grad-CAM Works

Grad-CAM (Gradient-weighted Class Activation Mapping) reveals **which regions of the image** the model found most suspicious.

```
Red   = High AI signal  (very suspicious)
Yellow = Medium signal  (somewhat suspicious)
Green  = Natural/Real   (looks authentic)
Blue   = No signal      (not suspicious)
```

It works by hooking into the last convolutional block of EfficientNet-B3, capturing gradients flowing back from the AI class score, and weighting the activation maps by those gradients to produce a spatial importance map.

## Limitations

- Trained on CIFAKE (Stable Diffusion images at 32×32px) — may miss newer generators like Midjourney v6 and DALL-E 3
- Images without EXIF data receive a baseline 40% metadata penalty even if real
- Video analysis averages frame scores without temporal modeling
